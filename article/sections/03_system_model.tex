\section{System Model}
\label{sec:system_model}

We consider a downlink \gls{dmimo} system with active \glspl{ap} $\mathcal{A}=\{1,\ldots,M\}$ and scheduled \glspl{ue} $\mathcal{U}=\{1,\ldots,K\}$. Let $N_m$ denote the number of transmit ports at \gls{ap} $m$, let $N_{r,k}$ be the number of receive ports at \gls{ue} $k$, and let $f \in \{1,\ldots,F\}$ index the sampled CFR tones over the $100\,\mathrm{MHz}$ band. The per-\gls{ap} channel block on tone $f$ is
\begin{equation}
    \mathbf{H}_{k,m}[f] \in \mathbb{C}^{N_{r,k} \times N_m},
\end{equation}
and the full distributed channel observed by \gls{ue} $k$ is
\begin{equation}
    \mathbf{H}_k[f] = \left[\mathbf{H}_{k,1}[f]\;\mathbf{H}_{k,2}[f]\;\cdots\;\mathbf{H}_{k,M}[f]\right].
\end{equation}

The antenna topology follows the \texttt{PlanarArray} abstraction used by the simulator. Each node is modeled as a \gls{upa} with $N^{\mathrm{row}}$ rows and $N^{\mathrm{col}}$ columns, so that the number of polarization-agnostic elements is $N^{\mathrm{row}}N^{\mathrm{col}}$. The implementation uses the same element radiation pattern, polarization setting, and inter-element spacing on transmit and receive arrays. More precisely, the number of ports is
\begin{equation}
    N = N^{\mathrm{row}} N^{\mathrm{col}} N^{\mathrm{pol}},
\end{equation}
where $N^{\mathrm{pol}}$ is the number of active polarization ports implied by the configured polarization string. In the Rabot configuration, the \glspl{ap} use a $2 \times 3$ \gls{upa} with \texttt{VH} dual polarization, which yields $N_m=2 \cdot 3 \cdot 2=12$ transmit ports per \gls{ap}. The \glspl{ue} use a $1 \times 1$ planar array with the same \texttt{VH} dual-polarized element model, which yields $N_{r,k}=1 \cdot 1 \cdot 2=2$ receive ports per user device. The vertical and horizontal element spacings are both set to $0.5\lambda$, so the array factor corresponds to half-wavelength sampling in both planar directions.

The element radiation pattern is the \gls{3gpp} \texttt{tr38901} pattern provided by Sionna. Thus, the simulated channel is not isotropic at the element level: every array response combines the \texttt{tr38901} element directivity with the \gls{upa} array factor determined by the node orientation, row-column topology, and the propagation geometry. This matters for the present study because the wall-mounted distributed \glspl{ap} and the rooftop central proxy are evaluated with the same underlying element pattern while differing in their locations, orientations, and distributed versus centralized array geometry.

For the central reference, the implementation collapses the distributed element budget into one normalized co-located rooftop array. If each distributed \gls{ap} has $N_{\mathrm{AP}}^{\mathrm{row}}N_{\mathrm{AP}}^{\mathrm{col}}$ physical element locations and the distributed budget contains $M_{\mathrm{dist}}$ \glspl{ap}, then the central proxy uses
\begin{equation}
    N_{\mathrm{cent}}^{\mathrm{elem}} =
    M_{\mathrm{dist}} N_{\mathrm{AP}}^{\mathrm{row}} N_{\mathrm{AP}}^{\mathrm{col}}
\end{equation}
element locations, reshaped into a factorized \gls{upa} with integer row and column counts. In the Rabot configuration, $M_{\mathrm{dist}}=6$ and the distributed hardware uses $2\times 3$ elements per \gls{ap}, so the normalized co-located rooftop proxy becomes a $6\times 6$ \gls{upa}. With the same \texttt{VH} dual polarization, this yields $6 \cdot 6 \cdot 2 = 72$ transmit ports. The central transmit-power budget is scaled to the sum power of the six distributed \glspl{ap}. This keeps the central reference aligned with the distributed element and power budgets, while still remaining a rooftop proxy rather than a full placement model for a true co-located base-station site.

Given a receive combiner $\mathbf{u}_k[f]$, the effective channel becomes
\begin{equation}
    \mathbf{h}_k^{\mathsf{H}}[f] = \mathbf{u}_k^{\mathsf{H}}[f]\mathbf{H}_k[f].
\end{equation}
Stacking all users yields the effective distributed channel matrix $\mathbf{H}_{\mathrm{eff}}[f]$. The current pipeline applies a least-squares pseudo-inverse \gls{zf} precoder independently on every sampled tone, followed by per-stream power allocation under a total \gls{ap} power budget.

For \gls{ue} $k$ on tone $f$, the instantaneous downlink \gls{sinr} is evaluated as
\begin{equation}
    \mathrm{SINR}^{\mathrm{DL}}_k[f] =
    \frac{p_k \left|\mathbf{h}_k^{\mathsf{H}}[f]\mathbf{f}_k[f]\right|^2}
    {\sum_{i \in \mathcal{U},\, i \neq k} p_i \left|\mathbf{h}_k^{\mathsf{H}}[f]\mathbf{f}_i[f]\right|^2 + \sigma^2},
\end{equation}
where $\mathbf{f}_i$ is the precoding vector for stream $i$, $p_i$ its allocated power, and $\sigma^2$ the thermal-noise term. The implementation keeps the interference term explicitly and exports desired-signal, interference, and noise power terms for each evaluated user snapshot.

The implementation evaluates this expression on all sampled tones and stores a scalar wideband \gls{sinr} per user by mapping the tone-averaged spectral efficiency back to a Shannon-equivalent \gls{sinr}. The \gls{esr} used later in the results section is derived from the same full-band per-snapshot \gls{sinr} samples. For snapshot $n$ with scheduled-user set $\mathcal{U}_n$, the postprocessing layer computes
\begin{equation}
    \mathrm{ESR}_n = \sum_{k \in \mathcal{U}_n}\frac{1}{F}\sum_{f=1}^{F}\log_2\!\left(1+\mathrm{SINR}^{\mathrm{DL}}_{k,n}[f]\right).
\end{equation}
These per-snapshot \gls{sinr} values form the basis of the trajectory-level \gls{sinr} and \gls{esr} statistics reported later in the article.

\subsection{Placement Strategies}

The placement layer compares four strategies under a common \gls{ap} budget:
\begin{itemize}
    \item[] \strategy1 -- one rooftop \gls{ap} is placed on the building whose rooftop representative point is nearest the area center; the \gls{bs} is oriented toward the area center. Its antenna topology is the normalized co-located array obtained by aggregating the distributed element budget into one rooftop \gls{upa}. 
    \item[] \strategy2 -- a distributed set of wall-mounted candidate positions is sampled once and kept fixed over the full trajectory.
    \item[] \strategy4 -- optimization 1. The same number of distributed \glspl{ap} is reselected at each relocation window from the wall-mounted candidate pool with a current-window nearest-user \gls{ue}-to-\gls{ue} proxy-\gls{csi} rule and a proxy-\gls{esr} objective.
    \item[] \strategy5 -- optimization 3. The same number of distributed \glspl{ap} is reselected at each relocation window from the wall-mounted candidate pool with the same nearest-user proxy association as optimization 1, but with a proxy-average-power objective based on $|\mathrm{CSI}|^2$.
\end{itemize}

Let $\mathcal{C}$ denote the wall-mounted candidate pool and let $\mathcal{S}_w \subseteq \mathcal{C}$ denote the selected movable-\gls{ap} subset in relocation window $w$.

\section{Relocation Heuristic}

The relocation mechanism repeats the same observe-score-relocate loop at every relocation instant. Optimization~1 and optimization~3 assume that the \gls{ap}-to-\gls{ue} channel of a dormant candidate position is not known before an \gls{ap} is actually moved there. Consequently, these two rules do not optimize directly over candidate-position \gls{ap}-to-\gls{ue} \gls{csi}. Instead, each candidate site is associated with the closest user within a distance threshold. Optimization~1 reuses the measured \gls{ue}-to-\gls{ue} wideband channel of that nearby user as a proxy and ranks the subset by proxy \gls{esr}. Optimization~3 uses the corresponding stored \gls{ue}-to-\gls{ue} link-power samples, scaled to the \gls{ap} transmit-power budget, and ranks the subset by proxy average power. Once the best subset has been chosen, the movable \glspl{ap} are relocated to those candidate positions and the actual deployed \gls{ap}-to-\gls{ue} \gls{csi} is recomputed for storage, postprocessing, and final reporting.

The movable distributed strategy is implemented as a windowed receding-horizon controller. Let $\{t_n\}_{n=0}^{N_t-1}$ denote the simulated snapshot times and let $T_{\mathrm{rel}}$ denote the relocation interval. The trajectory is partitioned into disjoint decision windows
\begin{equation}
    \mathcal{W}_w = \left\{n \in \{0,\ldots,N_t-1\} :
    \left\lfloor \frac{t_n-t_0}{T_{\mathrm{rel}}}\right\rfloor = w\right\}.
\end{equation}
At the beginning of each window $w$, the controller selects a subset $\mathcal{S}_w \subseteq \mathcal{C}$ with cardinality $\lvert \mathcal{S}_w \rvert = N_{\mathrm{mov}}$, where $N_{\mathrm{mov}}$ is the movable-\gls{ap} budget. The fixed distributed mode uses one time-invariant subset, whereas the movable mode recomputes $\mathcal{S}_w$ at every relocation instant.

\subsection{\strategy4[title] -- current-window nearest-user proxy-\gls{csi} controller.}
Let $d_{\mathrm{th}}$ denote the configured distance threshold. For each candidate site $c \in \mathcal{C}$ and snapshot $n \in \mathcal{W}_w$, define the anchor user
\begin{equation}
    \ell(c,n)=
    \arg\min_{u \in \mathcal{U}}
    \left\|\mathbf{r}_{u,n}-\mathbf{x}_c\right\|_2,
\end{equation}
and the validity indicator
\begin{equation}
    \delta(c,n)=
    \mathbf{1}\!\left\{
        \left\|\mathbf{r}_{\ell(c,n),n}-\mathbf{x}_c\right\|_2 \le d_{\mathrm{th}}
    \right\}.
\end{equation}
The \gls{ap}-to-\gls{ue} channels for the candidate site is \gls{csi} is not available before relocation. Therefore, Optimization~1 replaces it by a proxy channel extracted from the measured \gls{ue}-to-\gls{ue} \gls{csi}. If $h^{\mathrm{UU}}_{u,v,n}[f]$ denotes the sampled \gls{ue}-to-\gls{ue} coefficient from transmitter-user $v$ to receiver-user $u$ at snapshot $n$ and tone $f$, the proxy coefficient used for candidate $c$ is
\begin{equation}
    \tilde{h}_{u,c,n}[f]=
    \begin{cases}
        h^{\mathrm{UU}}_{u,\ell(c,n),n}[f], & \delta(c,n)=1,\; u \neq \ell(c,n), \\
        h^{\mathrm{UU}}_{v^\star,\ell(c,n),n}[f], & \delta(c,n)=1,\; u = \ell(c,n), \\
        0, & \delta(c,n)=0,
    \end{cases}
\end{equation}
where $v^\star=\arg\max_{v \in \mathcal{U}} \max_f |h^{\mathrm{UU}}_{v,\ell(c,n),n}[f]|$ supplies a surrogate self-link because the diagonal user-to-itself measurement is not observable. For a candidate subset $\mathcal{S}$, the proxy channel matrix $\tilde{\mathbf{H}}_n[f](\mathcal{S})$ is formed by stacking the columns $\tilde{h}_{u,c,n}[f]$ for all $c \in \mathcal{S}$. The same \gls{zf} procedure as in the deployed \gls{ap} case is then applied on every sampled tone, yielding a proxy \gls{sinr} $\tilde{\gamma}_{u,n}[f](\mathcal{S})$ and the current-window proxy-\gls{esr} objective
\begin{equation}
    J_w^{(1)}(\mathcal{S}) =
    \frac{1}{|\mathcal{W}_w|F}
    \sum_{n \in \mathcal{W}_w}
    \sum_{f=1}^{F}
    \sum_{u \in \mathcal{U}}
    \log_2\!\left(1+\tilde{\gamma}_{u,n}[f](\mathcal{S})\right).
\end{equation}
% Consequently, Optimization~1 uses measured \gls{ue}-to-\gls{ue} proxy \gls{csi} to rank candidate subsets, while avoiding the unrealistic assumption that dormant candidate-position \gls{ap}-to-\gls{ue} channels are already known.

\subsection{\strategy5[title] -- current-window nearest-user proxy-power controller.}
The proxy-power rule reuses the same nearest-user anchor $\ell(c,n)$ and validity indicator $\delta(c,n)$ as Optimization~1, but it does not use the complex proxy coefficient directly. Instead, it operates on the stored \gls{ue}-to-\gls{ue} link-power tensor. Let $\rho^{\mathrm{UU}}_{a,u,n}$ denote the ordered link-power entry returned by the simulator between anchor-user index $a$ and user index $u$ at snapshot $n$. The implementation forms the per-candidate proxy-power row
\begin{equation}
    \bar{\rho}_{u,c,n} =
    \begin{cases}
        \rho^{\mathrm{UU}}_{\ell(c,n),u,n}, & \delta(c,n)=1,\; u \neq \ell(c,n), \\
        \max_{v \in \mathcal{U},\, v\neq \ell(c,n)} \rho^{\mathrm{UU}}_{\ell(c,n),v,n}, & \delta(c,n)=1,\; u = \ell(c,n), \\
        0, & \delta(c,n)=0,
    \end{cases}
\end{equation}
and then rescales that row to the distributed-\gls{ap} transmit-power budget according to
\begin{equation}
    \tilde{\rho}_{u,c,n} =
    \frac{P_{\mathrm{AP}}}{P_{\mathrm{UE}}}\bar{\rho}_{u,c,n}.
\end{equation}
The optimization-3 objective is then
\begin{equation}
    J_w^{(3)}(\mathcal{S}) =
    \frac{1}{|\mathcal{W}_w|}
    \sum_{n \in \mathcal{W}_w}
    \frac{1}{|\mathcal{U}|}
    \sum_{u \in \mathcal{U}}
    \sum_{c \in \mathcal{S}}
    \tilde{\rho}_{u,c,n}.
\end{equation}
Optimization~3 is therefore the power-only counterpart of Optimization~1: it uses the same nearest-user proxy association and the same self-link substitution rule, but it requires only stored proxy link power rather than the complex proxy channel.

\subsection{AP Relocation Selection -- Greedy subset construction.}
Both proxy-based optimization rules use the same forward-selection search. Starting from $\mathcal{S}_w^{(0)}=\varnothing$, the controller adds one candidate at a time according to
\begin{equation}
    c_r^\star =
    \arg\max_{c \in \mathcal{C}\setminus \mathcal{S}_w^{(r-1)}}
    J_w^{(q)}\!\left(\mathcal{S}_w^{(r-1)} \cup \{c\}\right),
\end{equation}
\begin{equation}
    q \in \{1,2\},
    \qquad
    \mathcal{S}_w^{(r)} =
    \mathcal{S}_w^{(r-1)} \cup \{c_r^\star\},
    \qquad r=1,\ldots,N_{\mathrm{mov}},
\end{equation}
and finally sets $\mathcal{S}_w=\mathcal{S}_w^{(N_{\mathrm{mov}})}$. It emulates as if the channel vectors form the anchor users, where the new channels for the \glspl{ap}. It uses thus a forward-selection heuristic and not an exhaustive combinatorial optimizer.

\subsection{Physically Relocating APs}

After either optimization rule has chosen the subset, the selected geometric candidate sites are converted into physical relocations of labeled movable \glspl{ap}. Let $\mathbf{p}_m^{(w-1)}$ denote the previous-window position of movable \gls{ap} $m$ and let $\mathbf{q}_j^{(w)}$ denote the coordinates of the candidates in the newly selected subset. The controller solves the linear assignment problem
\begin{equation}
    \pi_w^\star =
    \arg\min_{\pi \in \Pi_{N_{\mathrm{mov}}}}
    \sum_{m=1}^{N_{\mathrm{mov}}}
    \left\|\mathbf{p}_m^{(w-1)}-\mathbf{q}_{\pi(m)}^{(w)}\right\|_2^2,
\end{equation}
where $\Pi_{N_{\mathrm{mov}}}$ is the permutation set on $\{1,\ldots,N_{\mathrm{mov}}\}$. This minimum-displacement matching preserves \gls{ap} identities across windows and yields schedule traces that correspond to physically meaningful AP trajectories instead of arbitrary candidate relabelings. This allows to investigate the required relocation distances, i.e., which strategies require the most relocations.
